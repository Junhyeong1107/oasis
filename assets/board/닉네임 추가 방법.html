로그인된 유저의 닉네임도 파이어스토어에 저장하려면, 해당 정보를 어딘가에서 가져와야 합니다. 아마도 닉네임은 로그인된 사용자의 정보와 관련이 있을 것입니다. 여기서는 firebase.auth().currentUser 객체를 사용하여 로그인된 사용자의 정보를 가져온 후, 해당 사용자의 닉네임을 파이어스토어에 함께 저장하도록 코드를 수정해 보겠습니다.

아래는 수정된 uploadImagesToFirebase 함수입니다:

jsx
const uploadImagesToFirebase = async () => {
  try {
    if (selectedImages.length == 0) {
      Alert.alert("Error", "이미지를 선택해주세요.");
      return;
    }

    const currentUser = auth.currentUser; // Get the logged-in user
    const userNickname = currentUser ? currentUser.displayName : ''; // Get the user's nickname

    const uploadPromises = selectedImages.map(async (selectedImage) => {
      const response = await fetch(selectedImage);
      const blob = await response.blob();
      const imageName = new Date().getTime().toString();
      const storageRef = await uploadImage(blob, imageName);

      const downloadURL = await getDownloadURL(storageRef);
      return downloadURL; // Return the URL for each uploaded image
    });

    const imageUrls = await Promise.all(uploadPromises);

    // Save the image URLs along with the user's nickname
    await saveImageDataToFirestore(imageUrls, activeButtons, userNickname);

    setSelectedImages([]);
    setActiveButtons([]); // Reset active button

    Alert.alert("Success", "이미지 업로드가 완료되었습니다.");
  } catch (error) {
    console.error("Error uploading image:", error);
    Alert.alert("Error", "이미지 업로드 중 오류가 발생했습니다.");
  }
};

또한 firebaseConfig.js 파일에서 saveImageDataToFirestore 함수를 수정해야 합니다. 해당 함수가 어떻게 정의되어 있는지는 코드에서 확인하지 못하고 있기 때문에, 이 부분은 제공된 정보를 기반으로 가정하겠습니다.

위의 코드 변경사항은 현재 로그인된 사용자의 닉네임을 가져와 saveImageDataToFirestore 함수에 인자로 전달하여 이미지와 함께 저장합니다. 필요한 경우 해당 함수를 수정하여 닉네임을 파이어스토어에 저장하도록 설정하세요.
